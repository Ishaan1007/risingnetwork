# Google Meet REST API Setup Guide

## Prerequisites
1. Google Cloud Project with Google Meet API enabled
2. OAuth 2.0 Credentials (Client ID + Client Secret)
3. Environment variables configured

## Step 1: Google Cloud Setup

1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Create new project or select existing
3. Enable APIs:
   - **Google Meet REST API** (not Calendar API)
   - **Google People API** (for user info, optional)
4. Create OAuth 2.0 Credentials:
   - Application type: Web application
   - Authorized redirect URIs: `http://localhost:3000/api/auth/google/callback`

## Step 2: Environment Variables

Add to `.env.local`:

```env
# Google Meet REST API
GOOGLE_CLIENT_ID=your_google_client_id_here
GOOGLE_CLIENT_SECRET=your_google_client_secret_here
GOOGLE_REDIRECT_URI=http://localhost:3000/api/auth/google/callback

# Optional: For development
NEXT_PUBLIC_GOOGLE_MEET_ENABLED=true

# Optional: For production (when you have real access tokens)
GOOGLE_ACCESS_TOKEN=your_real_access_token_here
```

## Step 3: Install Required Package

```bash
npm install googleapis
```

## Step 4: API Integration

The system uses:
- **Google Meet REST API** directly (not Calendar)
- **OAuth 2.0 flow** for authentication
- **Real Meet links** generated by Google
- **Meeting space management** for organization

## Features Available

✅ **Real Google Meet Links**
✅ **Meeting Scheduling** 
✅ **Participant Management**
✅ **Meeting Space Creation**
✅ **Professional Conference Integration**

## API Endpoints Used

- `POST /api/auth/google` - OAuth authentication
- `POST /api/meetings/schedule` - Create meetings
- `GET /api/meetings/schedule` - List meetings

## Security Notes

- Store credentials securely
- Use environment variables
- Never commit secrets to Git
- Enable HTTPS in production

## Development Notes

- Uses mock token if `GOOGLE_ACCESS_TOKEN` is not set
- Falls back to mock links if Google API fails
- Full error handling and logging
